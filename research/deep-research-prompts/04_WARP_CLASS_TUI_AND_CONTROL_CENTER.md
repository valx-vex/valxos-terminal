# Master Prompt: Warp-Class TUI And Control Center Research

You are researching the UX and systems design for `VALXOS Terminal`.

Target outcome:
- A TUI-first AI control center
- Closer in ambition to Warp as a developer cockpit, but centered on AI workflows
- Not just a chat panel: a command center for rules, sessions, scripts, MCP, extensions, agents, tasks, and model routing

Research questions:
- What makes Warp compelling as a control center, not just a terminal?
- Which UX patterns from Warp, terminal.shop, power-user TUIs, IDE sidebars, command palettes, and ops dashboards are worth adapting?
- What are the best interaction patterns for switching models, viewing context state, launching commands, managing rules, managing MCP servers, and browsing session artifacts?
- How should a TUI handle dense information without becoming chaos?
- Which parts should stay TUI-native and which parts may want a desktop or browser companion later?
- How can the interface feel fast, serious, and power-user friendly without becoming cryptic?

Deliverables:
- UX principles for VALXOS
- Information architecture
- Control center feature map
- Panel/layout recommendations
- Navigation model
- Session and artifact browsing model
- Rules/MCP/script manager design recommendations
- MVP TUI layout and future desktop evolution

Output format:
- Markdown
- Include concrete layout proposals
- Include a `Do This / Avoid This` section
- Include source links and screenshots references where relevant
